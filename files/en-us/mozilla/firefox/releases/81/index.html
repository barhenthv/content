This article provides information about the changes in Firefox 81 that will affect developers. Firefox 81 was released on September 22, 2020.

Changes for web developers

Developer Tools

No changes

Console

Debugger

Other tools

HTML

  • The <iframe> element's sandbox attribute now supports the allow-downloads token. ({{bug(1558394)}})

Removals

  • Support for the non-standard mozallowfullscreen attribute has been removed from <iframe>. Consider using allow="fullscreen" instead. ({{bug(1657599)}}) See Firefox Site Compatibility notes.

CSS

  • The value of clip for the {{CSSxRef("overflow")}} property is now supported, renaming overflow:-moz-hidden-unscrollable. ({{bug(1531609)}})
  • The {{CSSxRef("text-combine-upright")}} property is made non-animatable to comply with the spec. ({{bug(1654195)}})

Removals

  • The non-standard {{CSSxRef("::-moz-focus-outer")}} pseudo-element was removed.

JavaScript

No changes

Removals

HTTP

  • Firefox now accepts non-standard Content-Disposition headers with a unquoted filename containing spaces. ({{bug(1440677)}})

Removals

APIs

No changes

DOM

  • Worker and SharedWorker now strictly enforce a JavaScript MIME type for the executed script. ({{bug(1569123)}})

Media, WebRTC, and Web Audio

Removals

WebAssembly

No changes

Removals

WebDriver conformance (Marionette)

  • Now the setWindowRect capability is true by default for all desktop applications (including Thunderbird), and false on Android for GeckoView ({{bug(1650872)}}).
  • Added Fission support for the following commands: WebDriver:SwitchToFrame, WebDriver:SwitchToParentFrame, WebDriver:GetCurrentURL. All Fission compatible commands are only available when marionette.actors.enabled is set to true.
  • Fixed the broken tracking of browsing contexts after opening a new window ({{bug(1661495)}}).
  • In case of failures WebDriver:SwitchToWindow always returns an unified NoSuchWindowError now ({{bug(1663429)}}).

Removals

  • WebDriver:GetActiveFrame has been removed, because it's not part of the WebDriver specification and is no longer used ({{bug(1659502)}}).

Changes for add-on developers

  • tabs.saveAsPDF() is now supported on macOS. ({{bug(1653354)}})
